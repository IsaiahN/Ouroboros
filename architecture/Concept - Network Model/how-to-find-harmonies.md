1. Exploratory Agents (Pioneers)
Role: Initial pattern discovery in unknown territory
Resonance Strategy: Minimal, periodic checks

Query pattern: Check for cross-domain matches only when local pattern exhibits unusual properties (high novelty, high success rate, or structural elegance)

Frequency: ~1% of decisions (when pattern stands out as "interesting")

Query scope: Limited to 2-3 most-relevant other domains (via domain tags)

Rationale: Explorers need speed; they're the "first detectors" who flag potentially resonant patterns

Example: An explorer in mathematics discovers a q-series with strange convergence. It checks: "Does physics have anything with similar growth rates?" → quick tag-based lookup, not full cross-domain scan.

2. Refinement Agents (Optimizers)
Role: Deepen understanding in specific domains
Resonance Strategy: Targeted, problem-focused

Query pattern: When stuck on optimization, ask: "Have other domains solved similar structural problems?"

Frequency: ~10% of decisions (during impasses or when seeking inspiration)

Query scope: Domain-specific analogies (e.g., "optimization in physics" → "optimization in biology")

Rationale: Refiners work deeply; they benefit most from cross-domain analogies

Example: A mathematics refiner stuck on modular transformation checks: "How do physicists handle symmetry breaking?" → query physics packages with "symmetry" tag.

3. Validation Agents
Role: Verify and cross-check patterns
Resonance Strategy: Systematic but sampled

Query pattern: For validation decisions, check: "Does this pattern contradict established knowledge in other domains?"

Frequency: ~30% of decisions (when validating novel claims)

Query scope: All domains but only for critical contradictions (consistency checking)

Rationale: Validators are the network's immune system; they need broad but targeted checks

Example: A validator assessing a new mock theta identity checks: "Does this violate known physical conservation laws?" → query physics constraint packages.

4. Analysis Agents
Role: Find meta-patterns and abstractions
Resonance Strategy: Continuous and comprehensive

Query pattern: Primary function is to detect cross-domain resonances

Frequency: 100% of decisions (this is their core job)

Query scope: Full cross-domain, but using cached resonance graphs not raw queries

Rationale: Analysts are the "resonance specialists"; they maintain the resonance maps

Example: An analyst continuously updates the resonance graph showing connections between mock theta functions, black hole entropy, and vertex algebras.

5. Generalist Agents
Role: Bridge domains and translate concepts
Resonance Strategy: Moderate, translation-focused

Query pattern: When encountering a concept, ask: "How is this expressed in other domains?"

Frequency: ~50% of decisions (their purpose is cross-domain translation)

Query scope: Selected domain pairs based on current translation tasks

Rationale: Generalists are the network's translators; they need bilingual, not multilingual, checks
